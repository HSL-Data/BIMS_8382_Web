About this course

This course introduces methods, tools, and software for reproducibly managing, manipulating, analyzing, and visualizing large-scale biomedical data. Specifically, the course introduces the R statistical computing environment and packages for manipulating and visualizing high-dimensional data, covers strategies for reproducible research, and culminates with analyses of real biomedical data using R and Bioconductor packages.

This is not a “Tool X” or “Software Y” class. We want you to take away from this series the ability to use an extremely powerful scientific computing environment (R) to do many of the things that you’ll do across study designs and disciplines – managing, manipulating, visualizing, and analyzing large, sometimes high-dimensional data. Whether that data is gene expression data from yeast, microbial genomics data from B. pertussis, public health data from Gapminder, RNA-seq data from humans, influenza outbreak data, movie preference trends from Netflix, or truck routing data from FedEx, you’ll need the same computational know-how and data literacy to do the same kinds of basic tasks in each. We might show you how to use specific tools here and there, but these are not important – you probably won’t be using the same specific software or methods 10 years from now, but you’ll still use the same underlying data and computational foundation. That is the point of this series – to arm you with a basic foundation, and more importantly, to enable you to figure out how to use this tool or that tool on your own, when you need to.

Course Schedule

Apr 1: Intro to R

This novice-level introduction is directed toward life scientists with little to no experience with statistical computing or bioinformatics. This interactive introduction will introduce the R statistical computing environment and RMarkdown for publishing reproducible reports.

Apr 6: RMarkdown and dplyr

This session covers some of the barriers to reproducible research and how to start to address some of those problems. We will further cover using RMarkdown and RStudio to combine text and executable code to create readable and shareable reports. In the process of writing RMarkdown documents we will also explore the dplyr package. The dplyr package in R will help us to effectively manipulate and conditionally compute summary statistics over subsets of a dataset containing many observations.

April 8: Data Visualization with ggplot2

This session will cover fundamental concepts for creating effective data visualization and will introduce tools and techniques for visualizing large, high-dimensional data using R. We will review fundamental concepts for visually displaying quantitative information, such as using series of small multiples, avoiding “chart-junk,” and maximizing the data-ink ratio. We will introduce the ggplot2 package for advanced high-dimensional visualization. We will cover the grammar of graphics (geoms, aesthetics, stats, and faceting), and using ggplot2 to create plots layer-by-layer. Upon completing this lesson, students will be able to use R to explore a high-dimensional dataset by faceting and scaling arbitrarily complex plots in small multiples.

April 13: Exploratory Data Analysis

This session will introduce the idea of “tidy data”, and will introduce R packages that enable data manipulation and plotting to explore a dataset and understand what steps must be taken to clean the data.

April 15: Advanced Exploratory Analysis

Riffing off of Tolstoy, Hadley Wickham, the writer of many of the R packages we will use, says “Like families, tidy datasets are all alike but every messy dataset is messy in its own way”. Upon completing this lesson, students will be able to use the tidyr, stringr, dplyr, and ggplot2 packages in R to clean and explore their data.

April 20: TidyTuesday Challenge 1

Each week RStudio releases a new dataset in a TidyTuesday challenge so we will each load the dataset, manipulate it as needed, and create a data visualization that tells a story. At the end of the session we will share our viz with each other and look at some other coders’ contributions on Twitter to get some fresh ideas.

April 22: Essential Statistics

This workshop will provide hands-on instruction and exercises covering basic statistical analyses in R. This will cover descriptive statistics, exploratory data analysis, t-tests, ANOVA, linear models, and methods for discrete variable analysis. We will cover hypothesis testing and assessing the assumptions of the tests.

April 27: Linear Regression

This session will focus solely on conducting linear regression analyses in R. It will begin with an overview of regression analysis and then move on to include fitting the model, investigating the residuals, and visualizing the models.

April 29: Multiple Regression

Here we will expand our knowledge of regression models by considering models with several predictors and learning how to choose the "best fit" model.

May 4: Logistic Regression and Predictive Analytics

For this session we will learn to model binary outcomes and utilize data science principles to create predictive models. We will follow the train, test, validate method to develop the best predictive model.

May 6: Dimension Reduction

In this session we will investigate techniques for inspecting, visualizing, and reducing dimensions for high-dimensional data. We will examine techniques like PCA, clustering, and heatmaps that are useful for any number of large datasets.

Student Presentations

So much of learning biomedical datascience will be up to you. You each are studying a different topic with varying computational needs. As such, we would like you to choose a package we did not cover at length in class and create a vignette demoing for the class what the package is for and a few of its functions. You will turn in a rendered PDF of your vignette, provide a 10min recorded presentation with a partner, and critique presentations of your peers. This assignment is worth 20% of your course grade. As an example of a vignette, please see UVA Library’s Clay Ford’s vignette of the pwr package

FAQ

What are the pre-requisites?

There are none! This course does not assume any knowledge of programming or using a command-line interface, but if you’ve have had any experience here, the content won’t come as so much of a shock. But don’t panic. Command-line interfaces and programming languages like R are incredibly powerful and will be utterly transformative on your research. There’s a learning curve, and it’s near-vertical in the beginning, but it’s surmountable and the payoff is worth it!

Do I need a laptop?

YES. You must have access to a computer on which you can install software. The class will be primarily live coding with activities for you to independently code. As such, ideally you would have 2 monitors allowing you to follow along with the instructor in one screen while you code in a second. Please follow the setup instructions prior to the workshop.
